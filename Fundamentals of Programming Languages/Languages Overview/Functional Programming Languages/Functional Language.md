# Functional Language
A [[Functional Language]] is a [[Fundamentals of Programming Languages/Languages Overview/Programming Language]] which goes about solving problems by continual recursive calls on data and using mathematical functions. Typically, unless for [[IO]], we do not tend to modify the underlying data in our program, which is what distinguishes this modality of programming from imperative programming. 

---
# Functions as First Class Citizens
In functional programming we say that functions are first class citizens which means that they can be passed to other functions much like other types in languages, i.e. integers, floats, etc. 

---
# Higher-Order Programming
Higher order programming takes advantage of the idea that we can set  variables as functions in algorithms. However, this is distinct from the idea of applying functional programming to more difficult problems.

# Map Function 
The map function can be used to perform iteration in a functional situation by assigning each index of a list to something we can map to with a function. 

Map takes in a [[Lambda Function]] and a list, and then iterates over it.  We can think of the map function as applying the function to every item in it. We do not necessarily need a lambda function, but it works out.
